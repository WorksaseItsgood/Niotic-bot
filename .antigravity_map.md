# 🤖 Antigravity Bot Architecture Map

Ce fichier aide Antigravity à s'orienter rapidement dans la structure du projet pour des modifications futures sans scanner l'entièreté du dossier.

## 📂 Organisation Générale

La racine se trouve sous `C:\Users\natha\Documents\GitHub\Niotic-bot`.

- `bot.js` -> `src/bot.js` : Loader d'événements et de commandes dynamique. Point d’entrée. 
- `config.js` -> `src/config.js` : Wrapper de la base de données persistante (`src/database.json`). Gère les logs, antiraid et whitelist.

## 🛡️ Systèmes Principaux

### 1. Anti-Raid (`src/utils/antiRaid.js`)
L'Anti-Raid enregistre dynamiquement des frappes en cache (MAP `guildId`):
- `handleBotAdd` : Utilisé dans `src/events/guildMemberAdd.js`. S'active si un bot rejoint le serveur; kick automatique s'il n'est pas whitelisté via le owner.
- `checkThreshold` : Écoute les logs d’audits pour bannir/dé-rank un membre si un seuil d'infraction rapide (10s) est dépassé. Ce composant est déclenché par :
  - `src/events/channelCreate.js`
  - `src/events/guildBanAdd.js`

### 2. Logs Multi-Niveaux (`src/utils/logger.js`)
Dépend de la configuration par guilde : `logsLevel: "normal" | "moyen" | "extreme"`.
Les logs sont automatiquement triés vers les bons salons : `mod-logs`, `user-logs`, `server-logs`, `message-logs`, `channel-logs`, `role-logs`, `raid-logs`.

### 3. Gestion d'événements et Commandes
Tous les événements Discord sont chargés dans le dossier `src/events/`.
- `interactionCreate.js` : Gère l'exécution des *SlashCommands* ET la logique des *Boutons* (dashboards interactifs comme `/raid`).
- `ready.js` : Enregistrement REST avec l'API Discord v10 pour injecter les 50 commandes.

## 🗃️ Liste et Emplacements des Commandes (`src/commands/`)

Un script initial (`setup_commands.js`) a généré les fondations de 50 commandes divisées en catégories :

| Catégorie | Emplacement Ex. | Commandes Couvertes |
|-----------|------------------|----------------------|
| **Modération** | `src/commands/mod/` | \`ban\`, \`kick\`, \`mute\`, \`unmute\`, \`warn\`, \`unwarn\`, \`timeout\`, \`untimeout\`, \`clear\`, \`lock\`, \`unlock\`, \`slowmode\`, \`freeze\`, \`unfreeze\`, \`nick\` |
| **Config** | `src/commands/config/` | \`raid\`, \`logs-level\`, \`whitelist-add\`, \`whitelist-remove\`, \`whitelist-list\`, \`alert-channel\`, \`log-channel\`, \`derank-delay\`, \`prefix\`, \`settings\`, \`reset-config\`, \`status\` |
| **Infos** | `src/commands/info/` | \`user-info\`, \`server-info\`, \`bot-stats\`, \`logs\`, \`warnings\`, \`bans\`, \`mutes\`, \`modqueue\`, \`help\`, \`about\` |
| **Utilitaires** | `src/commands/utility/` | \`ping\`, \`uptime\`, \`report\`, \`suggest\`, \`invite\`, \`avatar\`, \`server-icon\`, \`role-info\`, \`channel-info\`, \`member-count\`, \`created-at\`, \`joined-at\`, \`test\` |

## ⚙️ Manipulation des Boutons
Toutes les interactions boutons renvoyées par `/raid` (Dashboard) et `/logs-level` sont hard codées et structurées dans le `if (interaction.isButton()) { ... }` du fichier `src/events/interactionCreate.js`.
